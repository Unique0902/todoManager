1. 루틴 자동 난이도 조정 추천 기능
일정 기간 수행률이 낮거나 높을 경우, AI 또는 조건 기반으로 난이도 상/하향 추천

예: "최근 2주간 평균 수행률 20% → 난이도 낮출까요?"

예: "최근 3주 연속 100% 수행 중 → 난이도 높일까요?"

2. 감정일기 감정 필터링/시각화 강화
mood_tag 기준으로 필터링, 감정 변화 추이 시각화 기능

감정 히트맵, 주간/월간 감정 그래프

사용자 정의 감정도 포함한 통계 가능

3. 루틴 수행 이력 상세 구조화
performed_dates를 단순 배열이 아닌 구조화된 기록으로 저장

예: [{ date: "2025-06-18", success: true, note: "좋은 컨디션" }, ...]

추후 수행 조건, 이유, 실패 이유 분석도 가능

발전 ==> 루틴 수행 이력 상세 구조화 (실패 기록 중심)
performed_dates는 단순 날짜 배열이 아닌 실패한 날만 별도로 기록

실패한 경우에만 실패 이유나 상태를 기록하도록 유도하면,
실패 패턴 분석이나 루틴 개선의 근거로 매우 유용함

json
복사
편집
failed_logs: [
  {
    date: "2025-06-20",
    reason: "너무 피곤해서 못 함",
    mood: "무기력",
    context: "전날 야근"
  },
  {
    date: "2025-06-23",
    reason: "잊어버림",
    mood: "바쁨",
    context: "출근길 일정 겹침"
  }
]
이 구조는 나중에 다음과 같은 기능으로 확장 가능:

실패 원인별 통계 (피로, 일정 충돌, 동기 저하 등)

감정/컨텍스트 기반 실패 예측 또는 알림

실패 이유별 개선 추천 (예: 알림 강화, 루틴 시간대 조정 등)

==> 

✅ 루틴 실패 기록 입력 방식 개선안
✅ 입력 시점:

하루 피드백 페이지에서 루틴 수행률 확인할 때

해당 루틴이 예정된 날인데 수행하지 않은 경우,
👉 "실패 이유 기록하기" 버튼 또는 팝업 표시

✅ 입력 항목 예시:

json
복사
편집
{
  "date": "2025-06-20",
  "routine_id": "abc123",
  "reason": "너무 피곤해서 못 함",
  "mood": "무기력",
  "context": "전날 야근"
}
✅ UI 흐름 예시:

하루 피드백 진입

오늘 예정된 루틴 리스트 표시 (수행 여부 체크 포함)

체크 안 된 항목은 → 실패 이유 입력 버튼 노출

입력한 실패 기록은 routine.failed_logs에 저장

🔄 연결 기능:

이후 루틴 상세페이지에서 실패 기록 타임라인 제공

분석 탭에서 실패 사유/감정별 분포 시각화 가능
